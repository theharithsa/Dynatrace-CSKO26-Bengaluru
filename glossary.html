Core Concepts

Digital Experience Monitoring (DEM)

The umbrella of Dynatrace capabilities spanning Real User Monitoring (RUM), Synthetic Monitoring, and Session Replay. DEM captures how real and simulated users experience your applications from end to end.

RUM (Real User Monitoring)

Passive monitoring of actual user interactions on web and mobile applications. Tracks load times, errors, geographies, devices, and Core Web Vitals to reveal lived performance.

Synthetic Monitoring

Active monitoring driven by scripted bots that simulate user journeys from chosen locations. Ideal for validating uptime and performance even when no real users are online.

Applications & Sessions

Application (RUM context)

A logical container representing a monitored web or mobile app. Defines domains, injection rules, and properties for capturing user actions.

User Session

A continuous sequence of actions from a single user. Starts with the first captured action and ends after inactivity or the session timeout (roughly 35 minutes by default).

User Action

A measurable interaction within a session such as a page load, button click, or tracked API call.

Load Action

A user action for a full page navigation. Includes DOM loading, resource timings, and Core Web Vital metrics.

XHR Action (AJAX/FETCH Action)

A user action triggered by background requests without a full reload. Tracks asynchronous calls like XHR or Fetch API.

Custom Action

A manually defined user action for interactions that do not automatically trigger network calls, such as opening modals.

Conversion Goals

Business KPIs configured in Dynatrace—like checkout completions or login success—measured across sessions and funnels.

Errors

Errors (in DEM)

User-visible failures detected at the frontend: failed requests, JavaScript runtime errors, and custom errors defined by your team.

Request Error

API or XHR call returning HTTP 4xx/5xx responses or failing due to timeouts and network issues.

JavaScript Errors

Browser runtime errors such as TypeError: undefined is not a function that affect user experience.

Custom Errors

Application-specific errors captured via the Dynatrace RUM API, for example form validation issues.

Mobile Application Crashes

Sudden termination of a mobile app captured by the Dynatrace mobile agent, complete with stack traces and device metadata.

User Data Enrichment

User Action Properties

Custom metadata—such as product IDs or cart values—attached to individual user actions.

User Session Properties

Metadata applied to entire sessions, like account tier, region, or subscription level.

User Tagging

Mapping anonymous session data to known identities such as usernames or emails.

Performance Metrics

W3C Timings

Browser-provided timings covering DNS lookup, TCP handshake, first byte, DOM load, and more throughout the page lifecycle.

Core Web Vitals

Key Google metrics for user experience:
  • LCP (Largest Contentful Paint) for load speed
  • FID (First Input Delay) for responsiveness
  • CLS (Cumulative Layout Shift) for visual stability

Apdex (Application Performance Index)

A satisfaction score based on response time thresholds, grading experiences as satisfied, tolerating, or frustrated.

Milestone Metrics

Timestamp milestones such as navigation start, DOM interactive, and first byte that help explain perceived load performance.

Tools & Traces

HAR File (HTTP Archive)

Browser-exported artifact detailing requests and responses for a page load—useful for deep performance analysis.

Fiddler Trace

Network capture recorded with Fiddler, helping compare frontend versus backend performance contributions.

PWAs (Progressive Web Apps)

Web apps with native-like features—offline use and installability—that may require specialized RUM injection approaches.

Session Replay

Visual replays of real sessions including clicks, scrolls, and DOM changes to accelerate UX troubleshooting.

Instrumentation

JS Injection

Automatic insertion of the Dynatrace RUM agent into HTML responses via OneAgent for effortless coverage.

JS Tag (Manual Injection)

A snippet manually added to the <head> of pages when auto-injection is not feasible, such as CDN-hosted apps.

Synthetic Tags

Labels applied to synthetic monitors to support grouping, filtering, and reporting.

Synthetic Monitoring Components

VU Controller (Virtual User Controller)

The orchestrator that schedules and coordinates execution of synthetic test scripts.

VU Player (Virtual User Player)

The engine that executes browser-based synthetic steps, acting like a robot user.

Browser Monitors

Synthetic monitors launching a real browser to simulate multi-step user journeys such as form fills.

HTTP Monitors (API Monitors)

Lightweight monitors issuing HTTP(S) requests to validate API availability, latency, and response codes.

Network Monitors

Synthetic probes measuring network performance, routing delay, packet loss, and availability from chosen locations.

Troubleshooting & Analysis

RUM Overload Protection

Safeguard that limits the number of user actions captured per minute during spikes to protect system health.

Session Splitting

Automatic division of very long sessions (over 200 actions) into multiple sessions for clarity.

Beacon

The data packet sent by the RUM agent to Dynatrace containing user action and session details.

CDN Monitoring

Strategies for tracking performance when content is delivered via Content Delivery Networks.

CORS Issues

Failures that occur when beacons or XHR requests are blocked by cross-origin restrictions.

CSP (Content Security Policy)

Browser security headers that can block the RUM agent unless the policy allows it.

Mobile DEM

Mobile RUM SDK

Dynatrace SDKs for iOS and Android capturing crashes, user actions, and network calls in native apps.

Mobile Custom Events

Developer-defined actions in mobile apps—for example, tapping a "Transfer Money" button.

App Version Tracking

Ability to segment performance data by app version or build for faster regression detection.

Device Context

Metadata about the mobile device—OS, model, orientation, and connectivity.

Advanced DEM

Business Events (BizEvents)

High-value business events streamed from RUM data for advanced analytics, automation, and alerting.

Synthetic Locations

Dynatrace points-of-presence around the globe where synthetic monitors execute.

Geo Heatmaps

Visualizations that reveal performance trends by geographic region.

User Flow Visualization

Session Flow charts that show how users progress between actions and where they drop off.

Error Rate Metrics

Aggregated percentage of failed actions compared to total actions.

Bounce Rate

Sessions where users perform only a single action before leaving.

First Party vs Third Party Resources

Classification of monitored content—owned by you versus loaded from external services.

JS API (dtrum)

Dynatrace-provided API for custom instrumentation: enter or leave actions, report errors, and add metadata properties.

Sampling

Intentionally limiting captured sessions to balance licensing and system performance.

Session Timeout

Configurable inactivity threshold—defaulting to about 35 minutes—after which a session closes.

Synthetic Recorder (Dynatrace Recorder)

Chrome extension for recording multi-step synthetic browser tests.

Additional & Refined Terms

Instrumentation (Enhanced Definition)

Embedding telemetry (JS agents, SDKs, hooks) into applications so Dynatrace can collect monitoring data. Without instrumentation, no RUM or tracing signals are captured.

New RUM Experience / Grail Storage

Dynatrace’s modern RUM ingestion path that stores data in Grail for unified schemas, flexible analytics, and improved scalability.

View (Mobile / New RUM)

A screen or window within mobile and new RUM contexts used to map transitions, measure load times, and associate errors.

ANR (Application Not Responding)

Mobile state where the main thread is blocked long enough for the OS to warn or terminate the app; Dynatrace captures ANRs on supported Android versions.

Native Crash Reporting (Mobile)

Capture of crashes originating from native code (for example C/C++ via Android NDK) when the app restarts shortly after the crash.

IdentifyUser / User Identification API

Dynatrace functions to map anonymous sessions to known user IDs, consolidating activity across devices and visits.

Business Events (Biz Events) – Enhanced

Business-level events—such as purchases or signups—that bridge observability metrics with commercial outcomes.

OpenTelemetry / OTLP / Propagators

Open standards for instrumentation across metrics, traces, and logs. Dynatrace can ingest OTLP data and understand propagated context.

Semantic Dictionary / Global Fields

Standardized Dynatrace field names—such as timestamp—that keep data aligned across logs, spans, metrics, and events.

Trace / Distributed Trace

The recorded path of a request as it hops across services, with each segment captured as a span that Dynatrace stitches together.

Davis Event / Problem / Root Cause (Davis AI)

Davis®, the Dynatrace AI, groups anomalies into problems and pinpoints root causes—distinct from individual user action errors.

Configuration API (RUM / Dynatrace API)

APIs that automate RUM settings including beacon domains and application detection rules.

Allowed Beacon Domains

Whitelists specifying which domains can receive beacon traffic; mismatches prevent data from reaching Dynatrace.

Content Security Policy / Traceparent / Beacon Blocking

Security policies, missing headers like traceparent, or firewalls can block RUM beacons—ensure required rules allow them.

Maximum User Action Duration / Clipping

Guardrail that closes or clips user actions exceeding Dynatrace duration limits to keep datasets clean.
